# FAMU Math Year Comic — Weekly Image Generation Budget Plan

## Goal
Prepare a sustainable, non-abusive image-generation workflow that produces enough reusable visual assets and comic panels to have the comic ready to launch in **August 2026**.

This plan assumes:
- **Peak hours:** about **20 images per 3-hour window**
- **Off-peak hours:** about **50 images per 3-hour window**
- We should stay **well below** those limits to avoid waste, throttling, or abuse-like behavior

---

# 1. Core Strategy

## Guiding Principle
Do **not** use image generation as if every panel must be made from scratch.

Instead, use a **three-layer workflow**:
1. **Reusable assets first**
2. **Panel generation second**
3. **Comic assembly and reuse third**

This reduces compute usage, improves character consistency, and lowers the risk of hitting limits.

## Priority Order
### Tier 1 — Highest Priority: Reusable Foundation Assets
These should be generated first because they can support dozens of future panels.

- Character reference sheets
- Character pose packs
- Expression sheets
- Duo compositions
- Recurring campus backgrounds
- Key landmark/venue backgrounds
- Important props

### Tier 2 — Medium Priority: High-Reuse Panel Types
These are scenes that will repeat throughout the comic year.

- Outdoor conversation table panels
- Walking campus panels
- Study-group panels
- Library/learning-commons panels
- Classroom reaction panels
- Dining/cafe panels

### Tier 3 — Lowest Priority: One-Off Story Panels
Only generate these when asset reuse cannot cover them.

- Special event scenes
- Unique camera angles
- emotional climax shots
- sports-specific crowd moments
- promotional cover-style scenes

---

# 2. Safe Weekly Image Budget

## Recommended Working Budget
To stay under the likely limits and avoid abuse patterns, use a conservative weekly budget.

### Peak-Hour Safe Budget
- **10 to 12 images per 3-hour session**
- no more than **1 peak session per day** unless truly necessary

### Off-Peak Safe Budget
- **20 to 25 images per 3-hour session**
- no more than **1 off-peak session per day** for routine work

## Weekly Operating Target
For safe ongoing production, target:
- **25 to 40 images per week** during regular building weeks
- **10 to 20 images per week** during revision/reuse weeks

This keeps the pace sustainable while leaving room for failed generations, retries, and corrections.

---

# 3. Weekly Production Rhythm

## Weekly Rhythm Overview
Use a repeating 4-part weekly cycle.

### Monday — Planning and Selection
- Review which scenes are needed next
- Decide which scenes can be built from existing assets
- Identify only the truly necessary new generations
- Prepare prompts before any generation begins

### Tuesday — Foundation/Asset Generation
- Generate highest-reuse assets first
- Prefer backgrounds, references, and compositions over one-off panels
- Save every successful generation into the repo immediately

### Wednesday — Panel Generation
- Generate only the week’s necessary comic panels
- Reuse existing assets whenever possible
- Stop before hitting suspected limits

### Thursday — Assembly and Layout
- Build pages from generated art
- Crop, reuse, reframe, and combine assets
- Add dialogue/caption planning

### Friday — Review and Repair
- Evaluate consistency
- Identify missing assets or weak panels
- Make a short list for next week rather than immediately generating more

### Weekend — Buffer / Off-Peak Opportunity
- If needed, use one conservative off-peak generation session
- Focus on backlog-clearing or high-value references only
- Avoid large bursts unless absolutely necessary

# 5. Comic Creation Workflow

## Step 1 — Build Weekly Storyboards
Each week of comic production should begin with:
- timeline review
- prose review
- storyboard page plan
- panel count estimate
- asset reuse map

## Step 2 — Mark Each Panel as One of Three Types
### Type 1: Reuse Panel
Can be assembled mostly from existing assets.
- Lowest cost
- Highest priority

### Type 2: Light Variation Panel
Needs a new render, but can be based heavily on existing poses/background types.
- Medium cost
- Acceptable when necessary

### Type 3: New Scene Panel
Needs a brand-new environment, composition, or emotional setup.
- Highest cost
- Use sparingly

## Step 3 — Build the Week Page-by-Page
For each page:
1. Determine if existing assets already solve it
2. Reuse background first
3. Reuse character pose second
4. Generate only the missing piece
5. Save final page assets and source pieces clearly in the repo

# 8. Practical Rules to Avoid Waste

1. **Never generate without a named purpose**
2. **Never generate a panel before checking if an existing asset can solve it**
3. **Save every successful generation immediately**
4. **Name files clearly for reuse**
5. **Do not burst-generate near the platform limit**
6. **Prefer off-peak sessions for larger batches**
7. **Treat image generation as a production budget, not an infinite tool**
8. **Stop after the first strong warning or failure**
